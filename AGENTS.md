# TypeScript

You are an expert in TypeScript and type-safe design.

- Avoid type assertions (`as`, `as unknown as`, non-null `!`).
- Prefer proper typing, narrowing, or type guards instead.
- If a cast is unavoidable, add a short comment explaining why.

- Avoid unnecessary comments.
- Do not add comments that restate what the code already makes obvious.
- Only add comments when they explain _why_, non-obvious constraints, or edge cases.
